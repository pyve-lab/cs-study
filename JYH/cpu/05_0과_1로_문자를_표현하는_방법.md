
## 1. 핵심 용어 정의

컴퓨터는 기본적으로 0과 1만 이해하지만, 다양한 문자를 입력하고 출력할 수 있습니다. 
이는 문자 집합, 인코딩, 디코딩이라는 세 가지 핵심 개념을 통해 가능합니다.

컴퓨터가 문자를 처리하기 위해 반드시 필요한 세 가지 개념입니다.

| 용어 | 의미 | 비고 |
| :--- | :--- | :--- |
| **문자 집합 (Character Set)** | 컴퓨터가 이해할 수 있는 문자의 모음(목록) | 목록에 없으면 표현 불가 |
| **인코딩 (Encoding)** | 문자를 0과 1(코드)로 변환하는 과정 | 사람의 문자 → 컴퓨터 코드 |
| **디코딩 (Decoding)** | 0과 1로 된 코드를 다시 문자로 변환하는 과정 | 컴퓨터 코드 → 사람의 문자 |

## 2. 아스키 코드 (ASCII Code)

가장 대표적이고 대중적인 문자 집합 및 인코딩 방법입니다.

* **대상:** 영어 알파벳, 숫자, 일부 특수 문자, 제어 문자.
* **구성:** 문자 하나당 **7비트** 사용 ($2^7 = 128$개 표현).
  * 실제로는 오류 검출용(패리티 비트)을 포함해 **1바이트(8비트)**로 저장됩니다.
* **한계:** 영어권 중심이라 한글, 한자 등 다양한 언어 표현이 불가능합니다.

## 3. 한글 인코딩: 완성형과 조합형

한글은 자음과 모음의 조합(초성+중성+종성)이라는 특수성이 있어 독자적인 방식이 발전했습니다.

### ① 인코딩 방식의 분류
* **완성형 (EUC-KR 등):** '가', '나'처럼 완성된 글자 마디마다 코드를 부여합니다.
* **조합형:** 초성, 중성, 종성 각각에 코드를 부여하여 결합합니다.

### ② EUC-KR
* **방식:** 대표적인 **완성형** 인코딩.
* **크기:** 글자당 **2바이트**를 사용합니다.
* **문제점:** 약 2,350개의 한글만 표현 가능하여, '쀍', '쀼' 같은 글자가 깨지는 현상이 발생합니다.
* 사이트 https://dencode.com/ 에서 인코딩을 직접 해볼 수 있습니다. Encoded에서 Hex String쪽을 확인 해 보면됩니다.

## 4. 유니코드 (Unicode)와 UTF-8

전 세계 모든 언어를 통합하여 표현하기 위한 현대의 표준 방식입니다.

### ① 유니코드 (문자 집합)
* 모든 언어, 특수 문자, **이모티콘**까지 포함하는 거대 통합 문자 집합.
* **코드 포인트:** 각 문자에 부여된 고유한 16진수 값 (예: `U+AC00`).
* 사이트 https://unicode-table.com/en/blocks/ 에서 각 문자나 기호 등에 대한 유니코드를 확인 할 수 있습니다.

### ② UTF-8 (인코딩 방식)
유니코드를 0과 1로 바꾸는 가장 대중적인 방법입니다.
* **가변 길이 인코딩:** 문자에 따라 **1~4바이트**로 크기가 변합니다.
  * 영어: 1바이트 / 한글: 3바이트 등
* **장점:** 아스키 코드와 호환되며 저장 공간을 효율적으로 사용합니다.
* 사이트 https://onlineutf8tools.com/convert-utf8-to-binary 에서 인코딩 값을 직접 확인 해볼 수 있습니다.

### 참고 : 글자가 깨지는 경우가 있다면?
인코딩이 호환되지 않는건 아닌지, 문자집합에 포함되지 않는 문자가 사용된 것은 아닌지 의심해 볼 수 있습니다.
