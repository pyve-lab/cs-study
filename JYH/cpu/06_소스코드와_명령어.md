
## 1. 언어의 분류: 고급 언어와 저급 언어

컴퓨터는 우리가 작성한 프로그래밍 언어를 그대로 이해할 수 없습니다. 
인간을 위한 '고급 언어'가 컴퓨터를 위한 '저급 언어'로 변환되는 과정과 그 원리를 정리합니다.

컴퓨터가 명령어를 처리하기 위해서는 언어의 변환 단계가 필요합니다.

| 분류 | 특징 | 예시 |
| :--- | :--- | :--- |
| **고급 언어 (High-Level Language)** | 사람이 읽고 쓰기 편하게 만들어진 언어 | C, C++, Java, Python 등 |
| **저급 언어 (Low-Level Language)** | 컴퓨터가 이해하고 실행하기 위해 만들어진 언어 (명령어) | 기계어, 어셈블리어 |

### ① 저급 언어의 종류
* **기계어 (Machine Code):** 0과 1로 이루어진 이진수 형태의 언어. 가독성을 위해 16진수로 표기하기도 함.
* **어셈블리어 (Assembly Language):** 0과 1로 된 기계어를 사람이 읽기 편한 형태로 번역한 언어. 한 줄 한 줄이 컴퓨터를 동작시키는 명령어와 대응됨.

## 2. 고급 언어가 변환되는 방식

고급 언어로 작성된 소스코드는 크게 두 가지 방식(컴파일, 인터프리트)을 통해 저급 언어로 변환되어 실행됩니다.

### ① 컴파일 언어 (Compile Language)
* **방식:** 컴파일러가 소스코드 **전체를 훑어** 한꺼번에 저급 언어(목적 코드)로 변환함.
* **특징:** 오류가 하나라도 있으면 컴파일 에러를 내뱉으며 실행되지 않음.
* **대표 언어:** C, C++ 등.
* **비유:** 책 전체를 한국어로 번역해서 전달하는 방식.

### ② 인터프리트 언어 (Interpreter Language)
* **방식:** 인터프리터에 의해 소스코드를 **한 줄씩** 즉시 저급 언어로 변환하여 실행함.
* **특징:** 소스코드 중간에 오류가 있어도 그 직전까지의 올바른 코드는 실행됨.
* **대표 언어:** Python 등.
* **비유:** 친구 옆에서 책 내용을 한 줄씩 읽어주는 방식.

### * 컴파일 & 인터프리트 과정 살펴보는 방법
소스코드는 컴퓨터 내부에서 바로 실행되는게 아니라, 컴퓨터가 알아들을 수 있는 저급언어 형태로 한번 변환된 다음에
컴퓨터는 실행됩니다.
사이트 https://godbolt.org/ 에서 고급언어가 저급언어로 어떻게 컴파일 되는지 확인 해볼 수 있습니다.

### * 모든 프로그래밍 언어가 컴파일 언어과 인터프리트 언어로만 구분되는 걸까?
컴파일 언어와 인터프리터 언어의 경계는 모호하며, 두 방식 모두 고급언어가 저급언어(기계어)로 변환되는 대표적인 방식으로 이해해야 합니다.
