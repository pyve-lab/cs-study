## 0. C언어의 컴파일 과정

사용자가 작성한 소스 코드가 컴퓨터가 실행할 수 있는 파일이 되기까지는 **전처리기 → 컴파일러 → 어셈블러 → 링커**라는 네 가지 단계를 거칩니다.

---

## 1. 컴파일 과정 4단계

### ① 전처리 (Preprocessing)
본격적인 컴파일 전의 준비 단계입니다.
* **주요 작업**: `#include`와 같은 지시문을 해석하여 외부 라이브러리나 소스 코드를 내 코드에 포함시키고, 매크로를 치환합니다.
* **결과물**: 전처리된 소스 코드 파일 (확장자: `.i`)

### ② 컴파일 (Compilation)
전처리가 완료된 소스 코드를 저급 언어로 변환하는 단계입니다.
* **주요 작업**: C언어와 같은 고급 언어를 CPU가 이해하기 쉬운 **어셈블리어**로 번역합니다.
* **결과물**: 어셈블리어 파일 (확장자: `.s`)

### ③ 어셈블 (Assemble)
어셈블리어를 실제 기계어로 변환하는 과정입니다.
* **주요 작업**: 어셈블러가 어셈블리어를 `0`과 `1`로 이루어진 기계어로 번역하여 **목적 코드**를 만듭니다.
* **결과물**: 목적 파일 (확장자: `.o`)
    * *주의: 목적 파일은 기계어로 되어 있지만, 아직은 실행이 불가능한 상태입니다.*

### ④ 링킹 (Linking)
여러 개의 목적 파일을 하나로 묶어 최종 실행 파일을 만드는 단계입니다.
* **주요 작업**: 여러 소스 파일 간의 연결이나 라이브러리 함수 호출을 서로 연결해 줍니다.
* **결과물**: 우리가 실제로 실행할 수 있는 **실행 파일** (확장자: `.exe` 등)



---

## 2. 주요 개념 비교

| 구분 | 목적 파일 (`.o`) | 실행 파일 (`.exe`) |
| :--- | :--- | :--- |
| **공통점** | 0과 1로 이루어진 기계어 | 0과 1로 이루어진 기계어 |
| **차이점** | 독립적으로 실행 불가능 | 링킹을 거쳐 즉시 실행 가능 |

---

## 3. 핵심 요약

1. **내부 과정의 분리:** 보통 컴파일러가 모든 과정을 한 번에 처리해 주지만, 내부적으로는 전처리-컴파일-어셈블-링킹의 순서를 따릅니다.
2. **목적 파일의 한계:** 기계어로 되어 있더라도 다른 소스 파일의 기능이나 라이브러리가 연결되지 않으면 실행 파일이 될 수 없습니다.
3. **확장자 흐름:** `.c` (소스) → `.i` (전처리) → `.s` (어셈블리어) → `.o` (목적 코드) → `실행 파일`
