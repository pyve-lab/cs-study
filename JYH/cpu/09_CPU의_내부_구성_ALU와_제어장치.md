
## 1. ALU (산술 논리 장치, Arithmetic Logic Unit)
ALU는 CPU 내부에서 실제 **계산(연산)**을 담당하는 장치입니다.

### ① ALU가 받아들이는 정보
* **피연산자 (Operand):** 연산의 대상이 되는 데이터로, `레지스터`로부터 전달받습니다.
* **제어 신호:** 제어장치로부터 어떤 연산(덧셈, 뺄셈 등)을 수행할지 지시를 받습니다.

### ② ALU가 내보내는 정보
* **연산 결과:** 숫자, 문자, 주소 등의 결과값을 `레지스터`로 보냅니다. 메모리가 아닌 레지스터에 먼저 저장하는 이유는 접근 속도가 훨씬 빠르기 때문입니다.
* **플래그 (Flag):** 연산 결과에 대한 부가 정보입니다. `플래그 레지스터`에 저장됩니다.

### **주요 플래그 종류**
| 플래그 종류 | 의미 | 사용 예시 |
| :--- | :--- | :--- |
| **부호 플래그** | 연산 결과의 부호를 나타냄 | `1`일 경우 음수, `0`일 경우 양수를 의미 |
| **제로 플래그** | 연산 결과가 `0`인지 여부를 나타냄 | `1`일 경우 결과가 `0`, `0`일 경우 `0`이 아님 |
| **캐리 플래그** | 올림수나 빌림수 발생 여부를 나타냄 | `1`일 경우 발생함, `0`일 경우 발생하지 않음 |
| **오버플로우 플래그** | 오버플로우 발생 여부를 나타냄 | `1`일 경우 발생함, `0`일 경우 발생하지 않음 |
| **인터럽트 플래그** | 인터럽트 가능 여부를 나타냄 | `1`일 경우 가능함, `0`일 경우 불가능함 |
| **슈퍼바이저 플래그** | 커널/사용자 모드 실행 여부를 나타냄 | `1`일 경우 커널 모드, `0`일 경우 사용자 모드 |

## 2. 제어장치 (Control Unit)
제어장치는 **명령어를 해석**하고 각 부품에 **제어 신호**를 보내 컴퓨터 전체를 조율합니다.

### ① 제어장치가 받아들이는 정보
* **클럭 (Clock):** 컴퓨터의 모든 부품이 박자에 맞춰 움직일 수 있게 하는 시간 단위 신호입니다.
* **해석할 명령어:** `명령어 레지스터`로부터 실행할 명령어를 전달받아 해석합니다.
* **플래그 값:** 연산 결과의 부가 정보를 참조하여 다음 명령 수행에 반영합니다.
* **외부 제어 신호:** 입출력 장치 등 CPU 외부에서 보내온 신호를 받아들입니다.

### ② 제어장치가 내보내는 정보 (제어 신호)
* **CPU 내부로 보내는 신호:**
    * `ALU`: 수행할 연산을 지시
    * `레지스터`: 데이터를 이동하거나 저장하도록 지시
* **CPU 외부로 보내는 신호:**
    * `메모리`: 데이터를 읽거나 쓰도록 지시
    * `입출력 장치`: 장치 작동, 읽기/쓰기, 상태 테스트 등을 지시

---

## 3. 핵심 요약
1. **ALU**는 레지스터에서 값을 가져와 계산하고, 결과와 **플래그**를 남깁니다.
2. **제어장치**는 클럭 박자에 맞춰 명령어를 해석하고, CPU 안팎의 부품들에 **제어 신호**를 뿌려주는 총지휘자 역할을 합니다.
3. **플래그**는 CPU가 연산 결과의 상태(음수 여부, 0 여부 등)를 파악하는 아주 중요한 단서가 됩니다.
