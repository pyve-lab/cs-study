# 장치 컨트롤러와 장치 드라이버

컴퓨터 내부(CPU, 메모리)와 다양한 외부 입출력장치가 소통하기 위한 핵심 매개체인 **장치 컨트롤러**와 **장치 드라이버**를 정리합니다.

---

## 1. 입출력장치 관리가 까다로운 이유
입출력장치는 CPU나 메모리에 비해 아래 두 가지 이유로 다루기가 어렵습니다.
1. **종류의 다양성**: 전 세계에는 수많은 장치가 있고, 장치마다 데이터 전송 방식이나 형식이 제각각이라 규격화하기 어렵습니다.
2. **전송률(Transfer Rate)의 차이**: CPU/메모리는 데이터 처리 속도가 매우 빠르지만, 대부분의 입출력장치는 상대적으로 매우 느립니다.

---

## 2. 장치 컨트롤러 (Device Controller)
입출력장치는 본체에 직접 연결되지 않고, 하드웨어 중계기인 **장치 컨트롤러**를 거쳐 연결됩니다.

### 1) 주요 역할
* **통신 중계**: CPU와 입출력장치 사이에서 '번역가' 역할을 수행하여 규격 차이를 극복합니다.
* **오류 검출**: 연결된 장치의 상태를 확인하고 오류가 있는지 체크합니다.
* **데이터 버퍼링**: 전송률이 높은 CPU와 낮은 장치 사이의 속도 차이를 완화하기 위해 데이터를 '버퍼(Buffer)'에 모았다가 한꺼번에 보냅니다.



### 2) 내부 구조 (핵심 레지스터)
장치 컨트롤러 내부에는 통신을 위한 3가지 핵심 레지스터가 있습니다.
* **데이터 레지스터**: CPU와 장치 사이에 주고받을 실제 데이터가 담기는 공간입니다.
* **상태 레지스터**: 장치가 입출력 준비가 되었는지, 작업 중인지, 오류가 있는지 등의 정보를 저장합니다.
* **제어 레지스터**: CPU가 장치에 특정 동작(읽기, 쓰기 등)을 지시하기 위한 정보가 담깁니다.

---

## 3. 장치 드라이버 (Device Driver)
장치 컨트롤러가 하드웨어적 통로라면, 이를 움직이게 하는 프로그램은 **장치 드라이버**입니다.

* **개념**: 장치 컨트롤러를 제어하고 동작시키는 **소프트웨어적인 통로**입니다.
* **특징**:
    * 새로운 하드웨어를 연결했을 때 드라이버가 설치되어야만 컴퓨터가 해당 장치를 인식합니다.
    * 운영체제(OS)는 이 드라이버를 통해 장치 컨트롤러와 소통하며 장치를 실행합니다.
    * 드라이버가 없다면 CPU는 장치 컨트롤러를 어떻게 조작해야 하는지 알 수 없습니다.



---

## 4. 요약 및 비교

| 구분 | 장치 컨트롤러 (Device Controller) | 장치 드라이버 (Device Driver) |
| :--- | :--- | :--- |
| **형태** | 하드웨어 (전자 회로판) | 소프트웨어 (컴퓨터 프로그램) |
| **위치** | 입출력장치 내부 또는 메인보드 | 운영체제 내부 (커널 영역) |
| **역할** | 물리적인 데이터 중계 및 버퍼링 | 하드웨어를 제어하는 논리적 방법 제공 |
