# RAID의 정의와 종류

여러 개의 물리적 보조기억장치를 마치 하나의 논리적 장치처럼 사용하여 **성능**을 높이거나 **안전성(신뢰성)**을 강화하는 기술인 **RAID**의 개념과 레벨별 특징을 정리합니다.

---

## 1. RAID의 정의 (Redundant Array of Independent Disks)
* **개념**: 독립적인 디스크(HDD, SSD 등) 여러 개를 하나로 묶어 관리하는 기술입니다.
* **목적**:
    1. **성능 향상**: 데이터를 분산 저장하여 읽기/쓰기 속도를 높입니다.
    2. **데이터 안전성**: 복사본이나 복구 정보를 저장하여 디스크 고장에 대비합니다.

---

## 2. 주요 RAID 레벨별 특징

### 1) RAID 0 (스트라이핑, Striping)
* **방식**: 데이터를 줄무늬(Stripe)처럼 나누어 여러 디스크에 번갈아 분산 저장합니다.
* **장점**: 여러 디스크가 동시에 작동하므로 **입출력 속도가 매우 빠릅니다.**
* **단점**: 디스크 중 하나만 고장 나도 전체 데이터를 잃게 되어 **안전성이 낮습니다.**

### 2) RAID 1 (미러링, Mirroring)
* **방식**: 데이터를 거울처럼 다른 디스크에 똑같이 복사하여 저장합니다.
* **장점**: **복구 및 백업이 매우 쉽고 안전**합니다.
* **단점**: 전체 용량의 절반만 사용할 수 있어 **비용이 비싸고**, 쓰기 속도가 다소 느립니다.

### 3) RAID 4 (패리티 디스크 사용)
* **방식**: 데이터는 분산 저장하되, 오류 검출 및 복구를 위한 **패리티(Parity) 정보**를 한 대의 전용 디스크에 몰아서 저장합니다.
* **장점**: RAID 1보다 적은 디스크로도 안전성을 확보할 수 있습니다.
* **단점**: 데이터를 쓸 때마다 패리티 디스크에 부하가 집중되어 **병목 현상**이 발생합니다.

### 4) RAID 5 (분산 패리티)
* **방식**: RAID 4의 병목 현상을 해결하기 위해 패리티 정보를 **모든 디스크에 분산하여 저장**합니다.
* **특징**: 성능과 안전성의 균형이 가장 좋아 실무에서 널리 쓰이는 방식입니다.

### 5) RAID 6 (이중 패리티)
* **방식**: RAID 5와 유사하지만, 서로 다른 종류의 패리티 정보를 **이중(두 개)으로 저장**합니다.
* **장점**: **디스크 2개가 동시에 고장 나도 데이터를 복구**할 수 있어 안전성이 매우 높습니다.
* **단점**: 패리티를 두 번 계산해야 하므로 쓰기 속도가 RAID 5보다 느릴 수 있습니다.

---

## 3. 요약 및 비교

| 레벨 | 주요 특징 | 핵심 장점 | 주요 단점 |
| :--- | :--- | :--- | :--- |
| **RAID 0** | 스트라이핑 | **압도적 속도** | 데이터 유실 위험 높음 |
| **RAID 1** | 미러링 | **높은 신뢰성** | 저장 용량 효율 낮음 |
| **RAID 4** | 전용 패리티 | 효율적 백업 | 패리티 디스크 병목 |
| **RAID 5** | 분산 패리티 | **성능/안전성 균형** | - |
| **RAID 6** | 이중 패리티 | **최상급 안전성** | 상대적으로 느린 쓰기 |

---

**결론**: 성능이 최우선이면 **RAID 0**, 비용 대비 효율과 안전이 중요하다면 **RAID 5**, 데이터 보안이 무엇보다 중요하다면 **RAID 1이나 6**를 선택하는 것이 최적입니다.
