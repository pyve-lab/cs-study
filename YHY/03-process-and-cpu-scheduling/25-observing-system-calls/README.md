## 1) 개요

- 유저 프로그램이 파일을 읽거나( `open` / `read` ), 화면에 출력하거나( `write` ), 프로세스를 생성하는 등  
  **OS 도움이 필요한 순간**에는 커널 기능을 직접 실행할 수 없어서 **시스템 호출(System Call)** 을 사용한다.
- 시스템 콜이 발생하면 **CPU가 커널 모드로 전환**되고, 커널 영역에 있는 코드가 실행된다.

---

## 2) 설명

- 시스템 콜을 호출하면 **(전통적으로는) 소프트웨어 인터럽트/트랩(trap)** 이 발생한다.
- 그 결과:
  1) 커널 모드로 전환  
  2) 커널의 시스템 콜 핸들러 진입  
  3) 커널이 요청 작업 수행 (파일 접근, 프로세스 정보 조회 등)  
  4) 결과를 유저 모드로 반환  

---

## 3) 실습: `strace` 로 시스템 콜 관찰하기 (Linux)

> `strace` 는 프로그램이 실행 중 호출하는 **시스템 콜 목록을 그대로 출력**해준다.

### ✅ 준비 명령어
- `strace`
- `ls`

### ✅ 실습 명령어
```bash
strace /bin/ls
