# 다양한 보조기억장치

## 1️⃣ 하드 디스크 (Hard Disk Drive, HDD)

### 🔹 하드 디스크란?
- 자기적인 방식으로 데이터를 저장하는 보조기억장치
- 데이터는 자석의 극(N극, S극) 상태로 저장됨
- 이러한 이유로 하드 디스크는 **자기 디스크의 일종**으로 분류됨

---

### 🔹 자기적인 방식이란?

#### 하드 디스크의 자기적인 방식
- 하드 디스크는 데이터를  
  → **자석 방향으로 기억하는 저장장치**라고 생각하면 됨

#### 자석에는 방향이 있다
- 자석은 항상 **N극 / S극 방향**을 가짐
- 방향에 따라 상태가 달라짐

#### 하드 디스크는 이 방식을 이용
- 플래터 표면은 **아주 작은 자석 알갱이들**로 덮여 있음
- 자석 알갱이의 방향
  - 한쪽 방향 → 0
  - 반대 방향 → 1
- 이를 통해 **0과 1을 구분하여 저장**

#### 자기적인 방식의 특징
- 전기를 계속 흘려서 유지하지 않음
- 자석 방향만 바꿔 두면 그대로 기억됨
- 전원이 꺼져도 데이터가 남아 있음

---

## 2️⃣ 하드 디스크의 주요 구성 요소

### 🔹 플래터 (Platter)
- 실질적으로 데이터가 저장되는 부분
- 자기 물질로 덮여 있음
- 수많은 N극과 S극이 저장되어 있으며  
  → 이 극의 상태가 0과 1의 역할을 수행

### 🔹 스핀들 (Spindle)
- 플래터를 회전시키는 구성 요소
- 회전 속도는 **RPM (Revolutions Per Minute)**  
  → 분당 회전수로 표현

### 🔹 헤드 (Head)
- 플래터를 대상으로 데이터를 읽고 쓰는 부품
- 플래터 표면에 **아주 미세하게 떠 있는 상태**로 동작
- 바늘처럼 생긴 형태

### 🔹 디스크 암 (Disk Arm)
- 헤드를 원하는 위치로 이동시키는 부품

---

## 3️⃣ 플래터의 데이터 저장 구조

### 🔹 트랙 (Track)
- 플래터를 여러 개의 동심원으로 나눈 것
- 그중 하나의 원이 트랙

### 🔹 섹터 (Sector)
- 하나의 트랙을 피자 조각처럼 나눈 단위
- **하드 디스크의 가장 작은 전송 단위**

### 🔹 실린더 (Cylinder)
- 여러 장의 플래터에서  
  같은 위치의 트랙을 수직으로 연결한 논리적 단위
- 원통 모양의 공간

---

## 4️⃣ 실린더 단위로 데이터를 저장하는 이유
- 연속된 정보는 보통 **한 실린더에 기록**
- 이유:
  - 디스크 암을 움직이지 않고도
  - 여러 플래터에서 연속적으로 데이터 접근 가능

### 🔸 예시
- 플래터 2장을 사용하는 하드 디스크에서  
  네 개 섹터에 걸쳐 데이터를 저장할 경우
  1. 1번 플래터 앞면  
  2. 1번 플래터 뒷면  
  3. 2번 플래터 앞면  
  4. 2번 플래터 뒷면  
  순서로 저장

---

## 5️⃣ 하드 디스크 접근 시간의 구성 요소

### 🔹 탐색 시간 (Seek Time)
- 헤드를 목표 트랙까지 이동시키는 시간
- 디스크 암이 움직이는 시간

### 🔹 회전 지연 (Rotational Latency)
- 원하는 섹터가 헤드 아래로 올 때까지
- 플래터가 회전하는 시간

### 🔹 전송 시간 (Transfer Time)
- 하드 디스크와 컴퓨터 사이에서
- 실제 데이터가 전송되는 시간

---

## 6️⃣ 헤드 구조에 따른 하드 디스크 구분
- 단일 헤드 디스크  
  - 이동 헤드 디스크
- 다중 헤드 디스크  
  - 고정 헤드 디스크

---

## 7️⃣ 플래시 메모리 (Flash Memory)

### 🔹 플래시 메모리란?
- 전기적으로 데이터를 읽고 쓰는 반도체 기반 저장장치
- USB, SD 카드, SSD 등이 모두 플래시 메모리 기반

### 🔹 보조기억장치로만 보기 어려운 이유
- ROM
- 우리가 일상적으로 사용하는 거의 모든 전자제품 내부에 내장
- 저장장치라기보다는  
  → **다양한 시스템에서 널리 사용되는 저장 기술**

---

## 8️⃣ NAND 플래시 메모리
- 대용량 저장 장치에 널리 사용
- NOR 플래시 메모리도 존재하지만  
  → 이 책에서는 별도 언급이 없는 한 NAND 플래시 메모리를 의미

---

## 9️⃣ 셀(Cell)과 플래시 메모리 종류

### 🔹 셀(Cell)
- 플래시 메모리에서 데이터를 저장하는 가장 작은 단위
- 셀이 모여 MB, GB, TB 단위의 저장장치가 됨
- 한 셀에 몇 비트를 저장하느냐에 따라 종류가 구분

### 🔹 셀 타입별 구분

| 타입 | 셀당 저장 비트 |
|---|---|
| SLC | 1비트 |
| MLC | 2비트 |
| TLC | 3비트 |

(QLC는 4비트지만, 여기서는 SLC / MLC / TLC 위주)

---

## 🔟 SLC / MLC / TLC 특징

### 🔹 SLC (Single-Level Cell)
- 한 셀로 2개의 정보(0,1) 표현
- 입출력 속도 빠름
- 수명 김 (수만 ~ 수십만 번 쓰기/삭제 가능)
- 단점: 용량 대비 가격이 비쌈

**비유**
- 혼자 사는 집  
- 준비는 빠르지만 주거 비용이 큼

---

### 🔹 MLC (Multi-Level Cell)
- 한 셀로 4개의 정보 표현
- SLC보다 속도와 수명은 낮음
- 대용량화에 유리
- 시중의 많은 플래시 저장장치가 MLC 타입

**비유**
- 같은 집 개수일 때  
- 한 집에 여러 명이 사는 구조

---

### 🔹 TLC (Triple-Level Cell)
- 한 셀로 8개의 정보 표현
- 대용량화에 매우 유리
- SLC, MLC보다 수명과 속도는 낮음
- 용량 대비 가격이 저렴

---

## 1️⃣1️⃣ 셀 타입 선택 정리
- 같은 용량이라도  
  → 셀 타입에 따라 수명·성능·가격이 다름
- 자주 쓰고 지우거나 높은 성능이 필요 → **SLC**
- 대용량 + 저렴한 가격 → **TLC**
- 그 중간 → **MLC**

---

## 1️⃣2️⃣ 플래시 메모리 내부 구조 단위
- 셀 → 페이지 → 블록 → 플레인 → 다이

---

## 1️⃣3️⃣ 플래시 메모리의 가장 큰 특징
- 읽기 / 쓰기: 페이지 단위
- 삭제: 블록 단위
- 단위가 서로 다르다는 점이 핵심 특징

---

## 1️⃣4️⃣ 페이지 상태

| 상태 | 의미 |
|---|---|
| FREE | 저장 가능한 상태 |
| VALID | 유효한 데이터 저장 |
| INVALID | 더 이상 유효하지 않은 데이터 |

- 플래시 메모리는 덮어쓰기 불가능
- VALID 상태의 페이지에는 새 데이터 저장 불가

---

## 1️⃣5️⃣ 가비지 컬렉션 (Garbage Collection)

### 🔹 문제 상황
- 페이지 단위로는 삭제 불가
- 수정 시:
  - 남아 있는 FREE 페이지에 새 데이터 저장
  - 기존 데이터는 INVALID 상태가 됨
- 결과적으로 쓰레기 값 증가

### 🔹 정리 방식
- 한 블록에서 **유효한 데이터만** 새로운 블록에 복사
- INVALID 데이터가 포함된 **기존 블록 전체 삭제**
