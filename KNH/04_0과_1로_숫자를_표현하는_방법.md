# 4강 0과 1로 숫자를 표현하는 방법

---

## 1. 왜 컴퓨터는 0과 1만 사용할까?

컴퓨터는 전기 신호로 동작한다.

- 전기가 있음 → 1
- 전기가 없음 → 0

이 두 상태만 안정적으로 구분할 수 있기 때문에

컴퓨터는 모든 정보를 **0과 1의 조합**으로 표현한다.

---

## 2. 이진수 (2진법)

### 2.1 이진수란?

- 사용하는 숫자: 0, 1
- 각 자릿수는 **2의 거듭제곱**을 의미한다
- 맨 오른쪽 자리는 항상 **2⁰**

### 예시

101(2)
= 1×2² + 0×2¹ + 1×2⁰
= 4 + 0 + 1
= 5

즉, 이진수 `101`은 10진수 `5`이다.

---

## 3. 자릿값 개념 (모든 진법 공통)

어떤 진법이든 자릿값 계산 방식은 동일하다.

- 맨 오른쪽 자리 → 진법의 0제곱
- 왼쪽으로 갈수록 지수가 1씩 증가

### 일반식

aₙ×kⁿ + aₙ₋₁×kⁿ⁻¹ + ... + a₁×k¹ + a₀×k⁰

(k: 진법)

---

## 4. 이진수의 한계

이진수는 컴퓨터에게는 편리하지만

사람에게는 읽고 쓰기 불편하다.

예를 들어,

- 255 (10진수)
- 11111111 (2진수)

이진수는 길어질수록

가독성이 급격히 떨어진다.

---

## 5. 16진수 (Hexadecimal)

### 5.1 16진수란?

- 사용하는 숫자: 0 ~ 9, A ~ F
- A = 10, B = 11, C = 12, D = 13, E = 14, F = 15
- 진법의 기준: 16

### 예시

1F(16)

= 1×16¹ + 15×16⁰

= 16 + 15

= 31

---

## 6. 왜 16진수를 사용할까? (2진수와의 관계)

### 핵심 이유

> 16진수 한 자리는 2진수 4자리와 정확히 대응된다.
> 

### 대응 관계

| 2진수 | 16진수 |
| --- | --- |
| 0000 | 0 |
| 0001 | 1 |
| 0010 | 2 |
| 0011 | 3 |
| 0100 | 4 |
| 0101 | 5 |
| 0110 | 6 |
| 0111 | 7 |
| 1000 | 8 |
| 1001 | 9 |
| 1010 | A |
| 1011 | B |
| 1100 | C |
| 1101 | D |
| 1110 | E |
| 1111 | F |

### 예시

2진수 : 0001 1111

16진수: 1    F

즉, 00011111(2) = 1F(16)

---

## 7. 16진수의 장점

### 1️⃣ 가독성 향상

- 2진수보다 훨씬 짧다
- 비트 단위 값을 한눈에 파악 가능

---

### 2️⃣ 비트 단위 표현에 적합

- 메모리 주소
- 비트마스크
- 색상 코드 (`#FFFFFF`)

모두 **16진수 기반 표현**을 사용한다.

---

### 3️⃣ 컴퓨터 내부 구조와 잘 맞음

- CPU와 메모리는 비트 단위로 동작
- 4비트 단위 묶음과 정확히 대응

---

## 8. 정리

- 컴퓨터는 전기 신호 때문에 0과 1만 사용한다
- 숫자는 이진수로 표현된다
- 이진수는 자릿값이 2의 거듭제곱이다
- 이진수는 길어지기 때문에 16진수를 함께 사용한다
- 16진수 한 자리는 이진수 4자리와 대응된다
