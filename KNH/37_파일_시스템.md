# 파일 시스템 (File System)

## 1. 파일 시스템이란?

- **파일 시스템(File System)** 이란
    
    운영체제가 **보조기억장치에 파일과 디렉터리를 저장하고 관리하는 방식**
    
- 파일을:
    - 어디에 저장할지
    - 어떻게 접근할지
    - 어떻게 관리할지를 결정

👉 운영체제마다 지원하는 파일 시스템이 다름

---

## 2. 파티셔닝(Partitioning)과 포매팅(Formatting)

### 2-1. 파티셔닝이란?

- 하나의 보조기억장치를
    - **여러 개의 논리적인 영역(파티션)** 으로 나누는 작업

예:

- 하나의 디스크를
    - C 드라이브
    - D 드라이브로 나누는 것

---

### 2-2. 파티셔닝의 목적

- 용도별 공간 분리
- 운영체제 / 데이터 분리
- 여러 운영체제 설치 가능

---

### 2-3. 포매팅이란?

- 파티션에
    - **파일 시스템을 설정하는 작업**
- 운영체제가 파일을
    - 저장하고
    - 찾고
    - 관리할 수 있도록 구조를 만드는 과정

👉 포매팅을 해야만

👉 파티션을 파일 저장 공간으로 사용 가능

---

### 2-4. 파티셔닝 vs 포매팅

| 구분 | 의미 |
| --- | --- |
| 파티셔닝 | 디스크를 나누는 작업 |
| 포매팅 | 나눈 공간에 파일 시스템을 만드는 작업 |

---

## 3. 파일 할당 방법

파일 시스템은

**파일을 보조기억장치에 어떻게 배치할지**를 결정해야 한다.

---

### 3-1. 연속 할당 (Contiguous Allocation)

### 개념

- 파일을
    - **연속된 블록**에 저장하는 방식

### 장점

- 접근 속도 빠름
- 구현이 단순

### 단점

- 외부 단편화 발생
- 파일 크기 변경 어려움

---

### 3-2. 연결 할당 (Linked Allocation)

### 개념

- 파일을
    - **불연속적인 블록**에 저장
- 각 블록이
    - 다음 블록의 위치를 가리킴

### 장점

- 외부 단편화 발생 ❌
- 파일 크기 변경 쉬움

### 단점

- 임의 접근(Random Access) 어려움
- 포인터 손상 시 파일 접근 불가

---

### 3-3. 색인 할당 (Indexed Allocation)

### 개념

- 파일의 모든 블록 주소를
    - **색인 블록(Index Block)** 에 저장
- 색인 블록을 통해
    - 파일의 모든 블록 위치 확인

### 장점

- 임의 접근 가능
- 외부 단편화 발생 ❌

### 단점

- 색인 블록을 위한 추가 공간 필요

---

## 4. 파일 시스템 살펴보기

### 4-1. 파일 시스템의 역할

- 파일과 디렉터리 관리
- 파일 할당 및 접근 관리
- 저장 공간 효율적 사용

---

### 4-2. 운영체제별 파일 시스템

- 운영체제마다
    - 서로 다른 파일 시스템 사용
- 하나의 운영체제가
    - 여러 파일 시스템을 지원하기도 함

👉 파일 시스템은

👉 운영체제와 매우 밀접한 관계

---

## 5. 저널링 파일 시스템 (Journaling File System)

### 5-1. 등장 배경

- 시스템 오류, 전원 차단 등으로
    - 파일 시스템이 손상될 수 있음
- 파일 작업 도중 문제가 발생하면
    - 데이터 일관성 깨짐

---

### 5-2. 저널링 파일 시스템이란?

- 파일 시스템에서 발생하는
    - **변경 사항을 기록(저널)** 해 두는 방식
- 실제 데이터 변경 전에
    - 변경 내용을 먼저 기록

---

### 5-3. 저널링의 장점

- 시스템 장애 발생 시
    - 빠른 복구 가능
- 파일 시스템 일관성 유지
- 검사 시간 단축

👉 현대 운영체제에서 널리 사용

---

## 6. 마운트 (Mount)

### 6-1. 마운트란?

- 특정 저장 장치를
    - **파일 시스템의 한 디렉터리에 연결**하는 작업
- 저장 장치를
    - 디렉터리처럼 사용 가능하게 함

---

### 6-2. 마운트의 의미

- 디스크, USB, 외장하드 등은
    - 마운트되어야 사용 가능
- 마운트되지 않은 장치는
    - 파일 시스템에서 접근 불가

---

### 6-3. 마운트 예시 개념

- `/mnt/usb`
- `/media/cdrom`

👉 특정 디렉터리를 기준으로

👉 저장 장치의 파일 시스템이 연결됨

---

## 7. 파일 시스템 전체 흐름 요약

- 디스크 준비
    1. 파티셔닝
    2. 포매팅
- 파일 저장 방식 결정
    - 연속 / 연결 / 색인 할당
- 파일 시스템 관리
    - 저널링으로 안정성 확보
- 저장 장치 사용
    - 마운트를 통해 접근

---

## 핵심 요약

- 파일 시스템은 파일 저장과 관리의 핵심
- 파티셔닝 → 포매팅 후 사용 가능
- 파일 할당 방식에 따라 성능과 효율 달라짐
- 저널링으로 장애 복구 강화
- 마운트를 통해 저장 장치를 디렉터리로 연결
