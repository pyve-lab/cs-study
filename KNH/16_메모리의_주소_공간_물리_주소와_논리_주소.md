# 메모리와 주소 공간

## 1️⃣ 물리 주소와 논리 주소

### 🔹 물리 주소 (Physical Address)
- **메모리(RAM)의 실제 주소**
- 메모리 하드웨어가 사용하는 주소
- 실제로 데이터가 저장된 위치를 의미

---

### 🔹 논리 주소 (Logical Address)
- **CPU와 프로그램이 사용하는 주소**
- 프로그램이 메모리에 접근할 때 사용하는 주소
- 실제 메모리 위치와는 다를 수 있음

---

### 🔹 정리
- 프로그램은 **논리 주소**만 사용
- 메모리는 **물리 주소**를 사용
- 논리 주소와 물리 주소는  
  → **서로 다를 수 있음**

---

## 2️⃣ 물리 주소와 논리 주소의 변환

### 🔹 주소 변환의 필요성
- 프로그램이 사용하는 논리 주소를  
  → 메모리가 이해할 수 있는 물리 주소로 바꿔야 함

---

### 🔹 MMU (Memory Management Unit)
- **논리 주소를 물리 주소로 변환하는 하드웨어**
- CPU와 메모리 사이에 위치

---

### 🔹 주소 변환 과정
1. CPU가 논리 주소를 생성
2. MMU가 해당 논리 주소를  
   → 물리 주소로 변환
3. 메모리는 변환된 물리 주소를 사용해 접근

---

### 🔹 특징
- 주소 변환은 **프로그램 실행 중에 자동으로 수행**
- 프로그램은  
  → 물리 주소를 알 필요가 없음

---

## 3️⃣ 메모리 보호

### 🔹 메모리 보호의 필요성
- 하나의 프로그램이  
  → 다른 프로그램의 메모리 영역을 침범하는 것을 방지
- 운영체제의 안정성을 유지하기 위함

---

### 🔹 메모리 보호 방식
- 논리 주소가  
  → **허용된 범위 내에 있는지 검사**
- 허용되지 않은 주소 접근 시  
  → 오류 발생

---

### 🔹 보호의 주체
- MMU가 주소 변환 과정에서  
  → 메모리 접근을 함께 검사

---

### 🔹 결과
- 프로그램은  
  → 자신에게 할당된 메모리 영역만 접근 가능
- 잘못된 메모리 접근으로부터  
  → 시스템을 보호

---

## ✅ 정리

- 물리 주소
  - 메모리의 실제 주소
- 논리 주소
  - 프로그램이 사용하는 주소
- MMU
  - 논리 주소를 물리 주소로 변환
- 메모리 보호
  - 잘못된 메모리 접근을 방지

---

## ✅ 핵심 요약
- 프로그램은 논리 주소만 사용
- 실제 메모리 접근은 물리 주소로 수행
- 논리 주소 → 물리 주소 변환은 MMU가 담당
- 메모리 보호는 시스템 안정성을 위한 필수 기능
