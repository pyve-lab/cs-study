# 캐시 메모리

## 1️⃣ 저장 장치 계층 구조

### 🔹 저장 장치 계층 구조란?
- 저장 장치들을  
  → **속도, 용량, 가격**에 따라 단계적으로 구성한 구조
- CPU와 가까울수록
  - 빠름
  - 용량이 작음
  - 가격이 비쌈
- CPU에서 멀어질수록
  - 느림
  - 용량이 큼
  - 가격이 저렴

---

### 🔹 저장 장치 계층 구조의 목적
- **빠른 속도와 큰 용량을 동시에 얻기 위함**
- 모든 데이터를 빠른 저장 장치에 둘 수 없기 때문에  
  → 계층 구조를 사용

---

### 🔹 대표적인 계층 구조
- 레지스터
- 캐시 메모리
- 주기억장치(RAM)
- 보조기억장치

---

## 2️⃣ 캐시 메모리

### 🔹 캐시 메모리란?
- CPU와 주기억장치(RAM) 사이에 위치한 메모리
- CPU가 자주 사용하는 데이터를  
  → **미리 저장해 두는 공간**

---

### 🔹 캐시 메모리의 목적
- CPU와 RAM 사이의 **속도 차이**를 줄이기 위함
- RAM에 직접 접근하는 횟수를 줄여  
  → 전체 성능 향상

---

### 🔹 캐시 메모리의 특징
- RAM보다 빠름
- 용량이 작음
- 가격이 비쌈

---

### 🔹 캐시 메모리의 동작
- CPU가 데이터를 요청
  - 캐시에 데이터가 있음 → **캐시 히트**
  - 캐시에 데이터가 없음 → **캐시 미스**
- 캐시 미스 발생 시  
  → RAM에서 데이터를 가져와 캐시에 저장

---

## 3️⃣ 참조 지역성의 원리 (Locality of Reference)

### 🔹 참조 지역성이란?
- 프로그램이 메모리에 접근할 때  
  → **특정 영역에 집중되는 경향**

---

### 🔹 시간 지역성 (Temporal Locality)
- 최근에 접근한 데이터는  
  → 다시 접근할 가능성이 높음
- 예:
  - 반복문
  - 변수 재사용

---

### 🔹 공간 지역성 (Spatial Locality)
- 어떤 메모리 위치에 접근하면  
  → 그 주변 위치도 접근할 가능성이 높음
- 예:
  - 배열
  - 순차적인 명령어 실행

---

### 🔹 캐시 메모리와 지역성
- 캐시 메모리는  
  → **참조 지역성의 원리를 이용**
- 자주 쓰이거나, 곧 쓰일 데이터를  
  → 캐시에 저장하여 성능 향상

---

## 4️⃣ 정리

- 저장 장치 계층 구조는  
  → 속도와 용량의 균형을 위한 구조
- 캐시 메모리는  
  → CPU와 RAM 사이의 속도 차이를 완화
- 캐시 메모리가 효과적인 이유는  
  → **참조 지역성의 원리** 때문

---

## ✅ 핵심 요약
- 캐시 메모리는 빠른 접근을 위한 중간 저장소
- 캐시 히트율이 높을수록 성능 향상
- 참조 지역성은
  - 시간 지역성
  - 공간 지역성
  으로 구분됨
