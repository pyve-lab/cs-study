# 7강 명령어의 구조와 주소 지정 방식

---

## 1. 명령어란 무엇인가

명령어는 **CPU가 직접 이해하고 실행하는 작업의 최소 단위**이다.

CPU는 프로그램을 실행할 때 명령어를 **하나씩 순서대로** 처리한다.

---

## 2. 명령어의 기본 구조

대부분의 명령어는 다음 두 부분으로 구성된다.

- **연산 코드 (Opcode)**
- **오퍼랜드 (Operand)**

---

## 3. 연산 코드 (Opcode)

연산 코드는 **CPU가 수행해야 할 작업의 종류**를 나타낸다.

예시:

- 덧셈
- 뺄셈
- 곱셈
- 메모리에서 값 읽기
- 메모리에 값 저장
- 프로그램 흐름 제어

연산 코드는 CPU에게 **“무엇을 할지”** 알려준다.

---

## 4. 오퍼랜드 (Operand)

오퍼랜드는 연산에 사용할 **데이터 또는 데이터의 위치 정보**를 나타낸다.

오퍼랜드에는 다음과 같은 정보가 들어갈 수 있다.

- 실제 데이터 값
- 데이터가 저장된 메모리 주소
- 레지스터 정보

> 즉, 오퍼랜드는 “**무엇을 대상으로 작업할지**”를 나타낸다.
> 

---

## 5. 오퍼랜드의 개수

명령어에 따라 오퍼랜드의 개수는 달라질 수 있다.

- 0-주소 명령어
- 1-주소 명령어
- 2-주소 명령어
- 3-주소 명령어

오퍼랜드의 개수가 많을수록 한 명령어가 처리할 수 있는 정보는 늘어나지만 명령어 길이는 길어진다.

---

## 6. 주소 지정 방식(Addressing Mode)

주소 지정 방식은 **오퍼랜드가 가리키는 데이터의 위치를 해석하는 방법**이다.

같은 명령어라도 주소 지정 방식에 따라 동작 방식이 달라질 수 있다.

---

## 7. 즉시 주소 지정 방식 (Immediate Addressing)

오퍼랜드에 **실제 데이터 값이 직접 포함**된 방식이다.

### 특징

- 추가 메모리 접근이 필요 없음
- 실행 속도가 빠름
- 표현할 수 있는 값의 크기에 제한이 있음

### 예시

ADD R1, 5

→ 레지스터 R1에 5를 더함

---

## 8. 직접 주소 지정 방식 (Direct Addressing)

오퍼랜드에 **메모리 주소가 직접 포함**된 방식이다.

### 특징

- 구조가 단순함
- 주소 공간에 제약이 있음

### 예시

LOAD R1, 100

→ 메모리 주소 100에 있는 값을 R1으로 가져옴

---

## 9. 간접 주소 지정 방식 (Indirect Addressing)

오퍼랜드가 **다른 주소를 가리키는 주소**를 담고 있는 방식이다.

### 특징

- 더 넓은 주소 공간 표현 가능
- 메모리 접근 횟수 증가로 속도 저하 가능

### 예시

LOAD R1, (100)

→ 100번지에 저장된 주소가 가리키는 값을 R1으로 가져옴

---

## 10. 레지스터 주소 지정 방식 (Register Addressing)

오퍼랜드에 **레지스터가 직접 지정**되는 방식이다.

### 특징

- 메모리 접근이 없어 매우 빠름
- 사용할 수 있는 레지스터 수에 제한 있음

### 예시

ADD R1, R2

→ R1과 R2의 값을 더함

---

## 11. 레지스터 간접 주소 지정 방식 (Register Indirect Addressing)

레지스터가 **메모리 주소를 저장**하고 있는 방식이다.

### 특징

- 간접 주소 지정 방식과 유사
- 메모리 접근 횟수는 1회

### 예시

LOAD R1, (R2)

→ R2가 가리키는 메모리 주소의 값을 R1으로 가져옴

---

## 12. 주소 지정 방식 비교

| 주소 지정 방식 | 데이터 위치 | 특징 |
| --- | --- | --- |
| 즉시 | 명령어 내부 | 빠름, 크기 제한 |
| 직접 | 메모리 | 구조 단순 |
| 간접 | 메모리 안의 주소 | 유연, 느릴 수 있음 |
| 레지스터 | 레지스터 | 매우 빠름 |
| 레지스터 간접 | 레지스터가 가리키는 메모리 | 효율적 |

---

## 13. 정리

- 명령어는 연산 코드와 오퍼랜드로 구성된다
- 연산 코드는 수행할 작업을 결정한다
- 오퍼랜드는 데이터 또는 데이터의 위치를 나타낸다
- 주소 지정 방식은 오퍼랜드를 해석하는 방법이다
- 주소 지정 방식에 따라 성능과 표현 범위가 달라진다
