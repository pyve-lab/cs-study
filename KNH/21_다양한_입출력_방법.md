# 다양한 입출력 방법

## 1️⃣ 프로그램 입출력 (Programmed I/O)

### 🔹 프로그램 입출력이란?
- CPU가 **입출력장치에 직접 명령을 내려**
  → 데이터를 주고받는 방식
- CPU가 장치의 상태를 계속 확인하며  
  → 입출력이 완료될 때까지 대기

---

### 🔹 특징
- 구조가 단순
- CPU가 입출력 작업에 계속 관여
- 입출력장치가 느릴 경우  
  → CPU가 비효율적으로 대기

---

## 2️⃣ 메모리 맵 입출력 (Memory-Mapped I/O)

### 🔹 메모리 맵 입출력이란?
- 입출력장치의 레지스터를  
  → **메모리 주소 공간에 포함**시키는 방식
- 메모리 접근 명령어로  
  → 입출력장치 접근 가능

---

### 🔹 특징
- 메모리 접근 방식과 입출력 방식이 동일
- 메모리와 입출력 주소 공간을 구분하지 않음

---

## 3️⃣ 고립형 입출력 (Isolated I/O)

### 🔹 고립형 입출력이란?
- 메모리 주소 공간과  
  → 입출력 주소 공간을 **분리**하는 방식
- 입출력 전용 명령어 사용

---

### 🔹 특징
- 메모리와 입출력 주소 공간이 분리됨
- 입출력 접근 시  
  → 전용 입출력 명령어 필요

---

## 4️⃣ 인터럽트 기반 입출력 (Interrupt-Driven I/O)

### 🔹 인터럽트 기반 입출력이란?
- 입출력장치가 작업을 마치면  
  → **CPU에게 인터럽트 신호를 보내는 방식**
- CPU는 입출력 완료를 기다리지 않음

---

### 🔹 특징
- CPU는 다른 작업을 수행 가능
- 입출력 완료 시  
  → 인터럽트를 통해 처리

---

## 5️⃣ DMA 입출력 (Direct Memory Access)

### 🔹 DMA 입출력이란?
- 입출력장치가  
  → **CPU를 거치지 않고 메모리에 직접 접근**하는 방식
- 대량의 데이터를 효율적으로 전송하기 위함

---

### 🔹 DMA 컨트롤러
- DMA 입출력을 담당하는 하드웨어
- CPU 대신  
  → 메모리와 입출력장치 간 데이터 전송 수행

---

## 6️⃣ DMA 입출력 과정

1. CPU가 DMA 컨트롤러에 작업 지시
2. DMA 컨트롤러가  
   → 입출력장치와 메모리 간 데이터 전송 수행
3. 전송이 완료되면  
   → DMA 컨트롤러가 CPU에게 인터럽트 발생

---

## 7️⃣ 입출력 버스 (I/O Bus)

### 🔹 입출력 버스란?
- CPU, 메모리, 입출력장치 사이에서  
  → **데이터를 전달하는 통로**

---

### 🔹 역할
- 입출력장치와  
  → 시스템 구성 요소들을 연결
- 입출력 신호와 데이터 전달

---

## 8️⃣ 정리

- 프로그램 입출력
  - CPU가 직접 입출력 수행
- 인터럽트 기반 입출력
  - 입출력 완료를 인터럽트로 통지
- DMA 입출력
  - CPU 개입 최소화
- 메모리 맵 입출력 / 고립형 입출력
  - 입출력 주소 공간 관리 방식의 차이

---

## ✅ 핵심 요약
- 입출력 방식은  
  → CPU 개입 정도에 따라 구분
- 대량 데이터 전송에는  
  → DMA 입출력이 효율적
- 입출력 버스는  
  → 장치 간 통신을 담당
