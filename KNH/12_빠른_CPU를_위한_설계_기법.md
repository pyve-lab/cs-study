# 빠른 CPU를 위한 설계 기법

## 1️⃣ 클럭 (Clock)

### 🔹 클럭이란?

- **CPU 내부에서 연산 타이밍을 맞추는 기준 신호**
- 클럭 신호가 한 번 반복될 때마다 CPU는 **아주 작은 작업 단위**를 수행
- 클럭 속도는 **1초에 클럭이 몇 번 반복되는지**를 의미

---

### 🔹 클럭의 단위

| 단위 | 의미 |
| --- | --- |
| Hz | 1초에 1번 |
| MHz | 1초에 1백만 번 |
| GHz | 1초에 10억 번 |

👉 예: **3.5GHz = 1초에 35억 번의 클럭 신호**

---

### 🔹 클럭 속도는 항상 일정하지 않다

현대 CPU는 **상황에 따라 클럭을 조절**한다.

### ✔ 기본 클럭 (Base Clock)

- CPU가 **안정적으로 유지할 수 있는 최소 보장 속도**
- 지속적인 작업에서도 유지 가능

### ✔ 최대 클럭 (Boost / Turbo Clock)

- 짧은 시간 동안 **조건이 허용되면 자동 상승**
- 발열·전력 여유가 있을 때만 가능

### ✔ 오버클럭킹 (Overclocking)

- 사용자가 **의도적으로 기본/최대 클럭을 초과**
- 성능 ↑ / 발열·전력·수명 ↓
- 모든 CPU가 지원하지는 않음

---

### 🔹 클럭 속도만 높이면 빨라질까?

❌ **아니다**

- 클럭 ↑ → 발열 ↑ → 한계 발생
- 명령어 구조, 병렬 처리 능력이 더 중요해짐
- 그래서 등장한 개념이 👉 **멀티코어**

---

## 2️⃣ 코어 (Core)

### 🔹 코어란?

- **명령어를 실제로 처리하는 CPU의 핵심 연산 단위**
- 과거에는 CPU = 코어 1개 (싱글코어)
- 현재는 **여러 개의 코어를 가진 멀티코어 프로세서**가 일반적

> 📌 용어 정리
> 
> 
> **멀티코어 프로세서** = 하나의 CPU 안에 여러 개의 코어가 존재
> 

---

### 🔹 싱글코어 → 도데카코어 정리

| 명칭 | 코어 수 |
| --- | --- |
| 싱글코어 | 1 |
| 듀얼코어 | 2 |
| 트리플코어 | 3 |
| 쿼드코어 | 4 |
| 헥사코어 | 6 |
| 옥타코어 | 8 |
| 데카코어 | 10 |
| 도데카코어 | 12 |

---

### 🔹 멀티코어의 장점

- 여러 작업을 **동시에 병렬 처리 가능**
- 멀티태스킹 성능 향상
- 전력 효율 개선 (클럭만 올리는 것보다 효율적)

---

### 🔹 ❗ 연산 속도는 코어 수에 비례하지 않는다

예:

- 2코어 = 2배 빠름 ❌
- 4코어 = 4배 빠름 ❌

### 이유

1. **프로그램이 병렬 처리를 지원해야 함**
2. 코어 간 **자원 공유로 인한 병목**
3. 스케줄링, 캐시 일관성 유지 비용
4. 일부 작업은 **직렬 처리**만 가능

스레드 (Thread)
1️⃣ 스레드란?

CPU에서 실행되는 가장 작은 실행 단위

하나의 프로그램은 하나 이상의 스레드를 가질 수 있음

CPU는 스레드를 단위로 명령어를 인출·실행

2️⃣ 하드웨어 스레드와 소프트웨어 스레드

책에서는 스레드를 다음 두 가지로 구분한다.

2-1️⃣ 하드웨어 스레드 (Hardware Thread)
🔹 개념

하나의 코어가 동시에 처리할 수 있는 스레드의 수

CPU 관점에서의 스레드

운영체제가 인식하는 실제 실행 단위

🔹 특징

하드웨어 스레드 수는 CPU 설계에 의해 결정

코어 하나가 여러 스레드를 동시에 실행할 수 있음

이 기술을 멀티스레드 프로세서라고 함

🔹 멀티스레드 프로세서

하나의 코어가 여러 개의 스레드를 처리

대표적인 구현 방식:

SMT (Simultaneous Multithreading)

🔹 목적

하나의 스레드가 대기 상태에 있을 때
→ 다른 스레드를 실행하여 자원 활용률 향상

2-2️⃣ 소프트웨어 스레드 (Software Thread)
🔹 개념

프로그램 또는 운영체제가 생성하는 스레드

논리적인 실행 흐름

🔹 특징

하나의 프로세스 안에 여러 개의 소프트웨어 스레드 존재 가능

실제 실행은 하드웨어 스레드 수에 의해 제한

3️⃣ 하드웨어 스레드와 소프트웨어 스레드의 관계
구분	설명
소프트웨어 스레드	프로그램이 만든 실행 흐름
하드웨어 스레드	CPU가 동시에 처리할 수 있는 실행 흐름

소프트웨어 스레드가 많아도
→ 하드웨어 스레드 수보다 많으면 번갈아 실행

4️⃣ 코어 수와 스레드 수의 관계

코어 수: 실제 연산 장치의 개수

하드웨어 스레드 수: 동시에 처리 가능한 실행 흐름 수

예:

4코어 / 8스레드 CPU
→ 코어 1개당 2개의 하드웨어 스레드

5️⃣ 핵심 정리 (책 기준 요약)

스레드는 CPU가 실행하는 최소 단위

스레드는

하드웨어 스레드

소프트웨어 스레드
로 구분

하드웨어 스레드는 CPU의 구조적 특성

소프트웨어 스레드는 프로그램이 만드는 실행 흐름

멀티스레드 프로세서는 CPU 자원 활용률을 높이기 위한 설계

이렇게 정리하면 책 흐름 그대로고,
앞에서 정리한 클럭 → 코어 → 스레드 구조도 딱 맞아.
