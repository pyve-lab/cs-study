# 장치 컨트롤러와 장치 드라이버

## 1️⃣ CPU - 입출력장치 정보 주고받기

### 🔹 문제 상황
- CPU와 메모리는  
  → 매우 빠르게 동작
- 입출력장치는  
  → 상대적으로 매우 느리게 동작

---

### 🔹 해결 필요성
- CPU가 입출력장치를 직접 제어하면  
  → 속도 차이로 인해 비효율 발생
- 따라서  
  → CPU와 입출력장치 사이를 중재하는 장치가 필요

---

## 2️⃣ 장치 컨트롤러 (Device Controller)

### 🔹 장치 컨트롤러란?
- **CPU와 입출력장치 사이에 위치한 하드웨어**
- 입출력장치를 제어하고  
  → CPU와의 통신을 담당

---

### 🔹 장치 컨트롤러의 필요성
- CPU와 입출력장치 간의  
  → 속도 차이를 완화
- 입출력장치의 동작을  
  → 표준화된 방식으로 처리

---

## 3️⃣ 장치 컨트롤러의 역할

- 입출력장치의 **동작 제어**
- CPU로부터 명령을 받아  
  → 장치에 전달
- 입출력장치의 상태를  
  → CPU에게 알림
- 데이터의 **버퍼링 수행**

---

## 4️⃣ 장치 컨트롤러의 구조

장치 컨트롤러는 일반적으로 다음 구성 요소를 가짐

---

### 🔹 데이터 레지스터
- CPU와 입출력장치 사이에서  
  → 데이터를 주고받는 공간

---

### 🔹 상태 레지스터
- 입출력장치의 현재 상태 저장
- 예:
  - 작업 중인지
  - 작업이 끝났는지
  - 오류가 발생했는지

---

### 🔹 제어 레지스터
- CPU가 입출력장치에게  
  → 수행할 작업을 지시

---

### 🔹 버퍼(Buffer)
- CPU와 입출력장치 간  
  → 속도 차이를 완화하기 위한 임시 저장 공간

---

## 5️⃣ 장치 드라이버 (Device Driver)

### 🔹 장치 드라이버란?
- **장치 컨트롤러를 제어하는 소프트웨어**
- 운영체제에 포함됨

---

### 🔹 역할
- 운영체제가  
  → 장치 컨트롤러를 사용할 수 있도록 도와줌
- 입출력장치마다  
  → 서로 다른 제어 방식을 처리

---

### 🔹 특징
- 같은 종류의 입출력장치라도  
  → 장치 드라이버가 다를 수 있음
- 운영체제는  
  → 장치 드라이버를 통해 장치를 제어

---

## 6️⃣ 정리

- CPU는 입출력장치를 직접 제어하지 않음
- 장치 컨트롤러
  - 하드웨어
  - CPU와 입출력장치 사이에서 중재
- 장치 드라이버
  - 소프트웨어
  - 장치 컨트롤러를 제어

---

## ✅ 핵심 요약
- 장치 컨트롤러는 하드웨어
- 장치 드라이버는 소프트웨어
- CPU ↔ 장치 컨트롤러 ↔ 입출력장치 구조로 동작
