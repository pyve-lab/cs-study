# CISC와 RISC

## 1️⃣ 명령어 집합 구조(ISA)

- CPU가 이해하고 실행할 수 있는 **명령어들의 집합**
- 명령어 집합 구조에 따라  
  → CPU의 설계 방식과 특성이 달라짐
- 대표적인 두 가지 방식
  - CISC
  - RISC

---

## 2️⃣ CISC (Complex Instruction Set Computer)

### 🔹 CISC란?
- **복잡한 명령어 집합**을 가진 구조
- 적은 수의 명령어로  
  → 많은 작업을 수행하도록 설계

---

### 🔹 특징
- 하나의 명령어가 **여러 작업을 수행**
- 명령어의 길이가 **가변적**
- 메모리에 직접 접근하는 명령어가 많음
- 프로그램의 **명령어 수를 줄이는 것**에 초점

---

### 🔹 장점
- 코드 길이가 짧아짐
- 복잡한 작업을 적은 명령어로 표현 가능

---

### 🔹 단점
- 명령어 해석과 실행이 복잡
- 파이프라이닝에 불리함
- 하드웨어 구조가 복잡해짐

---

## 3️⃣ RISC (Reduced Instruction Set Computer)

### 🔹 RISC란?
- **단순한 명령어 집합**을 가진 구조
- 명령어 하나당  
  → 하나의 작업 수행을 목표로 설계

---

### 🔹 특징
- 명령어의 길이가 **고정적**
- 대부분의 명령어가 **레지스터 간 연산**
- 메모리 접근은  
  → 로드(load), 스토어(store) 명령어로만 수행
- 명령어 수는 많아질 수 있음

---

### 🔹 장점
- 명령어 구조가 단순
- 파이프라이닝에 유리
- 하드웨어 설계가 상대적으로 단순

---

### 🔹 단점
- 동일한 작업을 수행하기 위해  
  → 더 많은 명령어 필요
- 코드 길이가 길어질 수 있음

---

## 4️⃣ CISC와 RISC 비교

| 구분 | CISC | RISC |
|---|---|---|
| 명령어 집합 | 복잡 | 단순 |
| 명령어 길이 | 가변 길이 | 고정 길이 |
| 명령어 수행 | 여러 작업 가능 | 한 작업 수행 |
| 메모리 접근 | 명령어에서 직접 접근 가능 | 로드/스토어 명령어로만 접근 |
| 파이프라이닝 | 불리 | 유리 |
| 하드웨어 구조 | 복잡 | 단순 |

---

## 5️⃣ 정리

- CISC와 RISC는  
  → **명령어 집합을 어떻게 설계할 것인가**에 대한 서로 다른 접근 방식
- CISC
  - 명령어 수를 줄이는 데 초점
- RISC
  - 명령어 구조를 단순화하는 데 초점
- 각각의 구조는  
  → CPU 설계 방식과 성능 특성에 영향을 줌

---

## ✅ 핵심 요약
- CISC: 복잡한 명령어, 적은 명령어 수
- RISC: 단순한 명령어, 파이프라이닝에 유리
- 두 방식 모두  
  → CPU 성능 향상을 위한 설계 방식
