# 6강 소스 코드와 명령어

---

## 1. 소스 코드와 명령어의 관계

우리가 작성하는 프로그램은 컴퓨터가 바로 이해할 수 있는 형태가 아니다.

- 사람이 작성하는 것: **소스 코드**
- 컴퓨터가 이해하는 것: **명령어**

즉,

컴퓨터는 소스 코드를 직접 실행하지 못하고 **명령어 형태로 변환된 결과만 실행**한다.

---

## 2. 소스 코드(Source Code)

소스 코드는 사람이 이해하기 쉽도록 작성된 프로그램 코드이다.

- 사람이 읽고 쓰기 쉬움
- 프로그래밍 언어로 작성됨
- 컴퓨터가 바로 실행할 수는 없음

예시:

int a = 1;

int b = 2;

int c = a + b;

이 코드는 사람이 보기엔 의미가 있지만, 컴퓨터 입장에서는 아직 실행할 수 없다.

---

## 3. 명령어(Instruction)

명령어는 CPU가 직접 이해하고 실행할 수 있는 최소 단위의 작업이다.

- 덧셈 수행
- 메모리에서 값 읽기
- 메모리에 값 저장
- 제어 흐름 변경

CPU는 **명령어를 하나씩 순서대로 실행**한다.

---

## 4. 소스 코드 → 명령어 변환 과정

소스 코드는 반드시 **명령어로 변환되는 과정을 거쳐야** 실행될 수 있다.

변환 방식은 소스 코드의 종류에 따라 다르다.

---

## 5. 고급 언어와 저급 언어

### 5.1 고급 언어 (High-level Language)

- 사람이 이해하기 쉬운 언어
- 추상화 수준이 높음
- 하드웨어 의존성이 낮음

예시:

- C
- Java
- Python

---

### 5.2 저급 언어 (Low-level Language)

- 컴퓨터가 이해하기 쉬운 언어
- 하드웨어와 밀접
- 사람이 이해하기 어려움

예시:

- 기계어
- 어셈블리어

---

## 6. 기계어(Machine Code)

기계어는

CPU가 직접 이해하는 **0과 1로 이루어진 명령어**이다.

- CPU는 기계어만 실행 가능
- 사람이 읽고 쓰기 매우 어려움

예시:

10101010 00001010

---

## 7. 어셈블리어(Assembly Language)

어셈블리어는

기계어를 사람이 이해하기 쉬운 **기호 형태**로 표현한 언어이다.

- 기계어와 1:1 대응
- CPU 구조에 강하게 의존

예시:

ADD R1, R2

---

## 8. 컴파일 언어와 인터프리터 언어

### 8.1 컴파일 언어

- 소스 코드를 **한 번에 명령어로 변환**
- 실행 파일 생성
- 실행 속도가 빠름

예시:

- C
- C++

---

### 8.2 인터프리터 언어

- 소스 코드를 **한 줄씩 해석하며 실행**
- 실행 파일 생성 없음
- 상대적으로 실행 속도가 느림

예시:

- Python
- JavaScript

---

## 9. 정리

- 사람은 소스 코드를 작성한다
- 컴퓨터는 명령어만 이해한다
- 소스 코드는 반드시 명령어로 변환되어야 실행된다
- 명령어는 CPU가 실행하는 최소 단위이다
- 기계어는 0과 1로 이루어진 명령어이다
- 어셈블리어는 기계어를 사람이 이해하기 쉽게 표현한 언어이다
- 고급 언어는 컴파일 또는 인터프리터 과정을 거쳐 명령어로 변환된다
