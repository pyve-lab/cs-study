# 9강 CPU의 내부 구성 – ALU와 제어장치

---

## 1. CPU 내부 구성 개요

CPU는 하나의 부품처럼 보이지만

실제로는 여러 내부 구성 요소가 협력하여 동작한다.

- ALU (산술논리연산장치)
- 레지스터
- 제어장치

이 장에서는 **ALU와 제어장치의 역할과 동작 방식**을 중심으로 정리한다.

---

## 2. ALU (산술논리연산장치)

ALU는 CPU 내부에서 **실제 연산을 수행하는 장치**이다.

### 2.1 ALU의 역할

ALU는 다음과 같은 연산을 담당한다.

- 산술 연산
    - 덧셈
    - 뺄셈
- 논리 연산
    - AND
    - OR
    - NOT
- 비교 연산
    - 두 값의 크기 비교
    - 같은지 여부 판단

즉, ALU는 CPU의 **계산기 역할**을 한다.

---

### 2.2 ALU의 입력과 출력

ALU는 단독으로 동작하지 않는다.

- **입력**
    - 레지스터에 저장된 값
- **출력**
    - 연산 결과
    - 연산 결과에 대한 상태 정보

연산 결과는 레지스터에 저장되거나 필요에 따라 메모리에 기록된다.

---

### 2.3 플래그 레지스터와 ALU

ALU는 연산을 수행한 후 연산 결과의 상태를 **플래그 레지스터**에 기록한다.

이 플래그 값들은 조건 분기나 CPU 제어 흐름에 사용된다.

> ALU는 계산을 수행하고, 계산 결과의 상태는 플래그로 남긴다.
> 

---

## 3. 제어장치 (Control Unit)

제어장치는 CPU 내부에서 **전체 동작을 조율하는 장치**이다.

### 3.1 제어장치의 역할

제어장치는 다음과 같은 역할을 수행한다.

- 명령어 해석
- 제어 신호 생성
- CPU 내부 구성 요소 제어
- 메모리와 입출력장치 제어

즉, 제어장치는 CPU의 **지휘자 역할**을 한다.

---

### 3.2 제어 신호란?

제어 신호는 컴퓨터의 각 부품을 작동시키기 위한 **전기 신호**이다.

예를 들어,

- 메모리에서 값을 읽을 때
→ 메모리 읽기 제어 신호
- 메모리에 값을 저장할 때
→ 메모리 쓰기 제어 신호
- ALU에 연산을 지시할 때
→ 연산 제어 신호

제어장치는 이러한 제어 신호를 **순서와 타이밍에 맞게** 발생시킨다.

---

## 4. 명령어 실행과 제어장치

CPU는 프로그램을 실행할 때 명령어를 **하나씩 순서대로 처리**한다.

이 과정에서 제어장치는 핵심적인 역할을 수행한다.

### 명령어 실행 흐름(개념)

1. 메모리에서 명령어를 가져온다
2. 명령어를 해석한다
3. 필요한 제어 신호를 생성한다
4. ALU 또는 다른 부품에 작업을 지시한다

> 제어장치는 “무엇을 언제 어떻게 할지”를 결정한다.
> 

---

## 5. ALU와 제어장치의 관계

ALU와 제어장치는 서로 역할이 명확히 구분되어 있다.

- **ALU**
    - 계산 담당
- **제어장치**
    - 계산 시점과 방식 결정

즉,

> 제어장치가 지시하고 ALU가 계산한다 라고 이해할 수 있다.
> 

---

## 6. 플래그 요약 표

플래그 레지스터에는 ALU 연산 결과와 CPU 제어에 필요한 정보가 저장된다.

| 플래그 | 의미 | 주요 사용 목적 |
| --- | --- | --- |
| 제로 플래그 | 결과가 0인지 여부 | 조건 분기 |
| 캐리 플래그 | 자리올림/내림 여부 | 부호 없는 연산 |
| 오버플로우 플래그 | 오버플로우 여부 | 부호 있는 연산 |
| 부호 플래그 | 결과의 부호 | 부호 판단 |
| 인터럽트 플래그 | 인터럽트 허용 여부 | 인터럽트 제어 |
| 슈퍼바이저 플래그 | 실행 권한 수준 | 운영체제 보호 |

---

## 7. 정리

- CPU는 여러 내부 구성 요소로 이루어진다
- ALU는 산술·논리 연산을 수행한다
- ALU는 연산 결과의 상태를 플래그로 남긴다
- 제어장치는 명령어를 해석하고 제어 신호를 발생시킨다
- 제어장치는 CPU 내부와 외부 장치의 동작을 조율한다
