# CPU 스케줄링 개요

## 1. CPU 스케줄링이란?

- **CPU 스케줄링(CPU Scheduling)** 이란
    
    여러 프로세스(또는 스레드)가 있을 때
    
    **어떤 프로세스에게 CPU를 할당할지 결정하는 것**
    

👉 CPU는 한정된 자원이기 때문에

운영체제가 **공정하고 효율적으로 배분**해야 한다.

---

## 2. CPU 스케줄링이 필요한 이유

- 동시에 실행 가능한 프로세스는 많음
- 하지만 CPU는:
    - 한 순간에 **하나의 프로세스만 실행 가능** (단일 CPU 기준)

👉 따라서 운영체제는

- “누가”
- “언제”
- “얼마나 오래”
    
    CPU를 사용할지 결정해야 한다.
    

---

## 3. CPU 스케줄링의 대상

- CPU 스케줄링의 대상은 **프로세스**가 아니라
    
    **스레드(Thread)**
    

👉 실제로 CPU에서 실행되는 단위는 **스레드**

---

## 4. 준비 큐(Ready Queue)

- **준비 상태(Ready)** 에 있는 프로세스(스레드)들은
- **준비 큐**에서 CPU 할당을 기다린다.

👉 스케줄러는

- 준비 큐에 있는 대상 중 하나를 선택해
- CPU를 할당한다.

---

## 5. CPU 스케줄러(CPU Scheduler)

- CPU 스케줄링을 담당하는 운영체제의 구성 요소
- 역할:
    - 준비 큐에서
    - 다음에 실행할 프로세스(스레드)를 선택

---

## 6. 디스패처(Dispatcher)

- 스케줄러가 선택한 프로세스를
- **실제로 CPU에 할당하는 역할**

디스패처가 수행하는 작업:

- 문맥 교환(Context Switch)
- 사용자 모드로 전환
- 선택된 프로세스 실행 시작

---

## 7. CPU 스케줄링이 발생하는 시점

CPU 스케줄링은 다음과 같은 상황에서 발생한다.

### 7-1. 실행 → 대기

- 실행 중이던 프로세스가
- 입출력 요청 등으로 대기 상태가 될 때

---

### 7-2. 실행 → 준비

- 타이머 인터럽트 발생
- CPU 사용 시간이 끝났을 때

---

### 7-3. 대기 → 준비

- 입출력 작업이 끝나
- 다시 실행 가능 상태가 될 때

---

### 7-4. 프로세스 종료

- 실행 중이던 프로세스가 종료될 때

---

## 8. 선점형 스케줄링과 비선점형 스케줄링

### 8-1. 비선점형 스케줄링 (Non-preemptive)

- 프로세스가 CPU를 할당받으면
- **스스로 CPU를 반납할 때까지 사용**

특징:

- 문맥 교환 횟수 적음
- 응답성이 떨어질 수 있음

---

### 8-2. 선점형 스케줄링 (Preemptive)

- 실행 중이던 프로세스라도
- **강제로 CPU를 빼앗길 수 있음**

특징:

- 타이머 인터럽트 사용
- 응답성이 좋음
- 문맥 교환 비용 발생

👉 책과 강의에서는

**현대 운영체제는 선점형 방식을 사용**한다고 설명

---

## 9. CPU 스케줄링의 목적 (평가 기준)

운영체제가 CPU 스케줄링을 할 때 고려하는 목표들:

- **CPU 이용률**
    - CPU를 최대한 쉬지 않게 사용
- **처리량(Throughput)**
    - 단위 시간당 완료된 작업 수
- **반환 시간(Turnaround Time)**
    - 작업 제출부터 종료까지 걸린 시간
- **대기 시간(Waiting Time)**
    - 준비 큐에서 기다린 시간
- **응답 시간(Response Time)**
    - 요청 후 처음 반응까지 걸린 시간

👉 이후 알고리즘들은

이 기준들을 얼마나 잘 만족하는지가 핵심

---

## 10. CPU 스케줄링 개요 정리

- CPU는 한정된 자원
- 여러 프로세스(스레드)가 CPU를 공유
- 운영체제가 스케줄링을 통해 관리
- 스케줄러가 대상 선택
- 디스패처가 실행
- 선점형 / 비선점형 방식 존재
- 성능 평가 기준 존재
