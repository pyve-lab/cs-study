# 운영체제의 큰 그림

## 1. 운영체제는 왜 필요한가?

컴퓨터는 다음과 같은 구성 요소를 가진다.

- CPU
- 메모리
- 보조기억장치
- 입출력 장치

이 하드웨어들은:

- 매우 빠르고
- 복잡하며
- 직접 다루기 어렵다

👉 운영체제는 이 복잡한 하드웨어를 **대신 관리**해 주는 소프트웨어다.

---

## 2. 운영체제의 핵심 역할

운영체제의 역할은 크게 두 가지로 요약된다.

### 2-1. 자원 관리 (Resource Management)

- CPU
- 메모리
- 저장 장치
- 입출력 장치

👉 여러 프로그램이 **한정된 자원**을 공정하고 효율적으로 사용하도록 관리

예:

- 어떤 프로세스가 CPU를 언제 사용할지
- 메모리를 어디까지 할당할지
- 파일을 동시에 접근해도 괜찮은지

---

### 2-2. 사용자 및 프로그램에 대한 서비스 제공

- 프로그램 실행
- 파일 입출력
- 네트워크 통신
- 오류 처리

👉 개발자는 하드웨어를 직접 다루지 않고

**운영체제가 제공하는 기능만 사용**하면 된다.

---

## 3. 운영체제는 어디에서 실행되는가?

### 3-1. 메모리 관점에서의 구분

운영체제는 메모리에서 다음 두 영역으로 나뉜다.

- 사용자 영역 (User Space)
- 커널 영역 (Kernel Space)

---

### 3-2. 커널(Kernel)이란?

- 운영체제의 **핵심**
- 가장 중요한 기능들이 실행되는 영역

커널이 담당하는 대표적인 기능:

- 프로세스 관리
- 메모리 관리
- 파일 시스템
- 입출력 장치 제어

👉 **운영체제 = 커널**이라고 생각해도 무방할 정도로 핵심적인 존재

---

### 3-3. 사용자 영역(User Space)

- 일반 응용 프로그램이 실행되는 공간
- 예:
    - 웹 브라우저
    - 게임
    - IDE
    - 메모장

👉 사용자 프로그램은 **하드웨어에 직접 접근할 수 없음**

---

## 4. 이중 모드(Dual Mode)

운영체제는 시스템을 보호하기 위해

**CPU를 두 가지 모드로 나누어 동작**시킨다.

### 4-1. 사용자 모드 (User Mode)

- 일반 프로그램이 실행되는 모드
- 제한 사항:
    - 하드웨어 직접 제어 ❌
    - 위험한 명령 실행 ❌

---

### 4-2. 커널 모드 (Kernel Mode)

- 운영체제가 실행되는 모드
- 모든 명령 실행 가능
- 하드웨어 접근 가능

---

### 4-3. 왜 이중 모드가 필요한가?

- 사용자 프로그램이
    - CPU를 독점하거나
    - 메모리를 마음대로 침범하거나
    - 시스템을 멈추는 행위를 막기 위해

👉 **시스템 안정성과 보안**을 위한 구조

---

## 5. 시스템 호출(System Call)

### 5-1. 시스템 호출이란?

- 사용자 프로그램이
- **운영체제의 기능을 사용하기 위해 요청하는 방법**

즉,

> ❝ 사용자 영역 → 커널 영역으로 넘어가는 공식적인 통로 ❞
> 

---

### 5-2. 시스템 호출의 흐름

1. 사용자 프로그램 실행 (사용자 모드)
2. 운영체제 기능이 필요함
3. 시스템 호출 발생
4. CPU가 커널 모드로 전환
5. 커널이 요청 처리
6. 다시 사용자 모드로 복귀

---

### 5-3. 시스템 호출 예시

- 파일 읽기 / 쓰기
- 프로세스 생성
- 네트워크 송수신
- 메모리 할당

👉 개발자가 직접 커널을 호출하는 게 아니라

**언어의 표준 라이브러리를 통해 간접적으로 사용**

---

## 6. 인터럽트(Interrupt)

### 6-1. 인터럽트란?

- CPU가 **현재 작업을 잠시 멈추고**
- 중요한 이벤트를 처리하도록 알리는 신호

---

### 6-2. 인터럽트의 종류 (책 기준 핵심)

### ① 하드웨어 인터럽트

- 입출력 장치
- 타이머
- 키보드, 마우스 등

예:

- 타이머 인터럽트 → CPU 사용 시간 종료 알림

---

### ② 소프트웨어 인터럽트

- 프로그램에 의해 발생
- 대표 예: **시스템 호출**

---

### 6-3. 인터럽트와 운영체제의 관계

- 운영체제는 **인터럽트를 중심으로 동작**
- 인터럽트가 발생하면:
    - CPU는 커널 모드로 전환
    - 운영체제가 제어권을 획득

---

## 7. 장치 드라이버(Device Driver)

### 7-1. 장치 드라이버란?

- 운영체제와 하드웨어 사이를 연결하는 **중개 소프트웨어**

---

### 7-2. 왜 필요한가?

- 하드웨어마다 제어 방식이 다름
- 운영체제가 모든 하드웨어를 직접 알 수 없음

👉 장치 드라이버가

**“이 장치는 이렇게 동작한다”** 를 운영체제에 알려줌

---

### 7-3. 구조 정리

```
응용 프로그램
   ↓
운영체제 (커널)
   ↓
장치 드라이버
   ↓
하드웨어

```

---

## 8. 운영체제의 큰 그림 요약

- 운영체제는 **자원 관리자**
- 커널이 운영체제의 핵심
- 사용자 영역과 커널 영역 분리
- 이중 모드로 시스템 보호
- 시스템 호출로 안전하게 커널 기능 사용
- 인터럽트 기반으로 동작
- 장치 드라이버로 하드웨어 제어
