# 파일과 디렉터리

## 1. 파일(File)

### 1-1. 파일이란?

- **파일(File)** 이란
    
    의미 있고 관련 있는 정보를 모아 놓은 **논리적 단위**
    
- 운영체제는
    - 보조기억장치에 저장된 데이터를
    - **파일 단위로 관리**

👉 사용자는 파일 단위로 데이터를 다루지만

👉 실제 저장 방식은 운영체제가 관리

---

### 1-2. 파일의 속성 (메타데이터)

파일에는 데이터뿐 아니라 **속성 정보**도 함께 저장된다.

대표적인 파일 속성:

- 파일 이름
- 파일 크기
- 생성 날짜 / 수정 날짜
- 소유자
- 접근 권한
- 파일 위치

👉 이런 정보를 **메타데이터(metadata)** 라고 함

---

### 1-3. 파일 연산

운영체제가 제공하는 대표적인 파일 연산:

- 생성(create)
- 삭제(delete)
- 열기(open)
- 닫기(close)
- 읽기(read)
- 쓰기(write)

👉 파일을 사용하기 위해서는

👉 보통 **열기(open)** → 작업 → **닫기(close)** 순서 필요

---

## 2. 디렉터리(Directory)

### 2-1. 디렉터리란?

- **디렉터리(Directory)** 는
    
    파일들을 관리하기 위한 **특별한 파일**
    
- 디렉터리 내부에는
    - 파일 정보
    - 다른 디렉터리 정보가 저장됨

👉 디렉터리도 파일의 한 종류

---

### 2-2. 디렉터리의 역할

- 파일을 **계층적으로 관리**
- 파일 검색과 관리를 쉽게 함
- 많은 파일을 효율적으로 관리 가능

---

### 2-3. 디렉터리 구조

- 대부분의 운영체제는
    - **트리 구조 디렉터리** 사용
- 하나의 최상위 디렉터리 존재
    - 이를 **루트 디렉터리(root directory)** 라고 함

---

## 3. 경로(Path)

### 3-1. 경로란?

- **경로(Path)** 란
    
    파일이나 디렉터리의 **위치를 나타내는 방법**
    

---

### 3-2. 절대 경로 (Absolute Path)

- **루트 디렉터리부터 시작**하여
- 파일 또는 디렉터리까지의 **전체 경로**

예시:

```
/home/user/document/file.txt

```

특징:

- 항상 같은 위치를 가리킴
- 기준이 명확

---

### 3-3. 상대 경로 (Relative Path)

- **현재 작업 중인 디렉터리**를 기준으로
- 파일 또는 디렉터리의 위치를 나타냄

예시:

```
document/file.txt

```

특징:

- 현재 위치에 따라 의미가 달라짐
- 경로가 짧아짐

---

## 4. 상대 경로를 나타내는 또 다른 방법

책과 강의에서는

상대 경로를 표현할 때 사용하는 **특별한 기호**를 함께 설명한다.

---

### 4-1. 현재 디렉터리: `.`

- `.` 은 **현재 디렉터리**를 의미

예시:

```
./file.txt

```

👉 현재 디렉터리에 있는 `file.txt`

---

### 4-2. 상위 디렉터리: `..`

- `..` 은 **부모(상위) 디렉터리**를 의미

예시:

```
../file.txt

```

👉 상위 디렉터리에 있는 `file.txt`

---

### 4-3. 활용 예시

현재 위치:

```
/home/user/project

```

- `./src/main.c`
    - `/home/user/project/src/main.c`
- `../document/report.txt`
    - `/home/user/document/report.txt`

---

## 5. 파일과 디렉터리 정리 요약

- 파일
    - 데이터 + 메타데이터
    - 운영체제가 파일 단위로 관리
- 디렉터리
    - 파일을 관리하는 특별한 파일
    - 계층 구조(트리 구조)
- 경로
    - 파일 위치 표현 방법
    - 절대 경로 / 상대 경로
- 상대 경로 표현
    - `.` : 현재 디렉터리
    - `..` : 상위 디렉터리
