# CPU 스케줄링 알고리즘

## 1. 선입 선처리 스케줄링 (FCFS)

### 개념

- **First Come First Served**
- 준비 큐에 **먼저 도착한 프로세스부터 실행**

### 특징

- 비선점형
- 구현이 단순함

### 단점

- **호위 효과(Convoy Effect)**
    - 긴 작업이 앞에 있으면 짧은 작업들이 오래 대기

---

## 2. 최단 작업 우선 스케줄링 (SJF)

### 개념

- **Shortest Job First**
- CPU 사용 시간이 **가장 짧은 프로세스부터 실행**

### 특징

- 평균 대기 시간 최소

### 단점

- 실행 시간 예측 어려움
- **기아 현상(Starvation)** 발생 가능

---

## 3. 최소 잔여 시간 우선 스케줄링 (SRT)

### 개념

- **Shortest Remaining Time**
- SJF의 **선점형 버전**
- 현재 실행 중인 프로세스보다
    - **남은 실행 시간이 더 짧은 프로세스가 오면 교체**

### 특징

- 선점형
- 평균 대기 시간 감소

### 단점

- 문맥 교환 잦음
- 실행 시간 예측 필요

---

## 4. 우선순위 스케줄링 (Priority Scheduling)

### 개념

- 각 프로세스에 **우선순위(priority)** 부여
- 우선순위가 높은 프로세스 먼저 실행

### 특징

- 선점형 / 비선점형 모두 가능
- 중요한 작업 먼저 처리 가능

### 단점

- 낮은 우선순위 프로세스의 **기아 현상**

---

## 5. 라운드 로빈 스케줄링 (RR)

### 개념

- **Round Robin**
- 모든 프로세스에게 동일한 **타임 퀀텀(Time Quantum)** 부여

### 동작

- 타임 퀀텀 종료 → 강제 선점 → 준비 큐 뒤로 이동

### 특징

- 선점형
- 응답성 좋음
- 대화형 시스템에 적합

### 단점

- 타임 퀀텀 설정이 중요
- 문맥 교환 비용 발생

---

## 6. 다단계 큐 스케줄링 (Multilevel Queue)

### 개념

- 준비 큐를 **여러 개의 큐로 분리**
- 각 큐는
    - 서로 다른 우선순위
    - 서로 다른 스케줄링 방식 사용

### 특징

- 큐 간 이동 ❌ (고정)
- 시스템 성격에 따라 작업 분류

### 단점

- 하위 큐 기아 현상 가능
- 유연성 부족

---

## 7. 다단계 피드백 큐 스케줄링 (Multilevel Feedback Queue)

### 개념

- 다단계 큐 + **피드백**
- 프로세스가 **큐 사이를 이동 가능**

### 특징

- CPU 사용 패턴에 따라 우선순위 조정
- 실행 시간이 길수록 아래 큐로 이동

### 장점

- 기아 현상 완화
- 매우 유연함

👉 **책에서 “가장 복잡하지만 가장 발전된 방식”으로 설명**

---

## 8. 7가지 알고리즘 한눈에 정리

| 알고리즘 | 선점 | 핵심 특징 |
| --- | --- | --- |
| FCFS | ❌ | 단순, 호위 효과 |
| SJF | ❌ | 평균 대기 시간 최소 |
| SRT | ⭕ | SJF 선점형 |
| Priority | ⭕/❌ | 우선순위 기반 |
| RR | ⭕ | 공정, 응답성 |
| Multilevel Queue | ⭕ | 큐 고정 |
| Multilevel Feedback Queue | ⭕ | 큐 이동 가능 |

---

## 책에서 강조하는 핵심 메시지

- **완벽한 스케줄링 알고리즘은 없다**
- 운영체제는
    - 성능
    - 공정성
    - 응답성
        
        사이에서 타협
        
- 실제 OS는
    - **여러 알고리즘을 조합**해 사용
